# Hotpot-Ordering-System
# Online Hotpot Reservation & Ordering System (線上火鍋訂位與點餐系統)

> **元智大學 資訊管理學系 大三課程專案 — 系統分析與設計**
> **全功能數位化餐飲服務系統，涵蓋從會員管理、線上訂位到後端點餐作業的完整流程。**

本專案旨在解決傳統餐飲業人工訂位與點餐的低效率問題，透過開發一套完整的 Web 應用程式，讓顧客可以隨時隨地預約座位與選擇餐點，並提供管理者高效的數據管理介面。

---

## 💡 核心功能 (Key Features)

* **數位化訂位服務**：顧客可線上選擇用餐日期、人數與預約時段，提升預約便利性。
* **完整點餐流程**：整合會員系統，顧客登入後可挑選湯底、肉類與副食，並自動生成訂單明細與總價計算。
* **雙權限入口管理**：
    * **會員端**：包含註冊、登入、個人訂位查詢與即時點餐功能。
    * **員工端/管理端**：提供後台查詢介面，可管理會員資料與所有交易明細，輔助營運決策。
* **前後端互動整合**：實現從前端輸入、資料庫儲存到後端查詢展示的完整互動流程。

---

## 🛠️ 技術堆疊 (Tech Stack)

* **Frontend**: `HTML5`, `CSS3`, `JavaScript` (UI/UX 設計與前端邏輯、RWD設計)
* **Backend**: `PHP` (伺服器端邏輯處理與資料交換)
* **Database**: `MySQL` (會員資料、商品資訊與訂單紀錄儲存)
* **System Design**: 包含完整的 `ERD` 規劃與 `系統架構圖` 設計。

---

## 🏗️ 系統功能模組 (System Modules)

1. **會員系統**：處理帳號註冊、登入驗證與資訊編輯。
2. **訂位模組**：管理座位預約時段，確保用餐排程不衝突。
3. **點餐模組**：串接商品資料庫，實現選餐、數量統計與訂單生成。
4. **管理系統**：提供管理員過濾與查詢全校交易資料的介面。

## 📚 專案收穫

透過本專案的開發，我深入掌握了以下核心技能：
* **資料庫設計**：從零開始規劃 ERD，確保資料庫表格間的關聯性 (Normalization) 與查詢效率。
* **前後端串接實作**：使用 PHP 處理顧客請求並與 MySQL 進行資料庫操作。
* **系統分析與設計**：將複雜的餐廳營運流程轉化為具體的系統功能模組。
